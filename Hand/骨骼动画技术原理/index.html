    骨骼动画技术原理¶

    众所周知，人物模型是由三角面 mesh 组成的。为了让人物动起来，一个最简单的想法是直接修改 mesh 的各个顶点的坐标，这带来了几个问题：

    1. 逐顶点操作很容易带来非常不自然的结果
    2. 随着模型越来越精细，顶点越来越多，操作空间十分巨大

    仿照自然界的生物体，一个自然的想法是，可以在 mesh 中放置若干骨骼，骨骼的运动带动 mesh 的运动。这样，动画师只需要操作骨骼，就可以带动 mesh 完成动画的编辑。

    由此，骨骼动画应运而生，它是目前游戏中最常用的人物动画技术。
    与骨骼对应的，mesh 被称为皮肤（skin）。
    一段段骨骼之间相互连接，连接处称之为关节（joint）。

    骨骼可以绕着关节进行转动。关节之间通过线段相连，可以形成一个树状结构，我们称之为关节模型：

    上图中，以腰部的关节为树根，其他关节根据连接关系形成了一棵树。

    工业界骨骼动画的制作流程¶

    1. 骨骼与标准姿势（T-pose）的绑定与蒙皮（binding, skinning）
    2. 动画师操作骨骼，制作关键帧（keyframe）
    3. 关键帧之间的插值形成整个动画

    绑定与蒙皮：对于标准姿势，创建一套骨骼用来对应 mesh 上的顶点。这个对应做好了，才可以完成驱动操作。另外，这个对应好不好直接影响了动画的质量，这也是让万千美术头秃的工作

    关键帧：这是制作动画的常用方法，通过指定关键的动作来表示动画

    插值：对于非关键帧的动画，通过插值来生成

    另一个角度上，可以这么理解：在关键帧编辑骨骼姿态，在非关键帧插值骨骼姿态，在所有帧上进行蒙皮。

    插值¶

    插值的目的是平滑连续。插值问题可以抽象成：给定\(f(0) =p_0, f(1) = p_1\) ，求 \(f(t), 0 < t< 1\)

    最简单的插值是线性插值，即 \(f(t) = tp_0 +(1-t)p_1\)

    如果要求更高的平滑性的话，我们可以再取端点的若干阶导数值，如 Hermite Curve 插值，其插值公式是
    \(f(t) = h_{00}(t)p_0 +h_{10}(t)m_0 + h_{01}(t)P_1 + h_{11}(t)m_1\)

    其中 \(p_0, p_1\) 是端点值， \(m_0, m_1\) 是端点导数值。四个参数如下计算
    $$
    \begin{align}
    h_{00}(t) = 2t^3 - 3t^2 + 1\
    h_{01}(t) = t^3 - 2t^2 + t\
    h_{10}(t) = -2t^3 + 3t^2\
    h_{11}(t) = t^3 - t^2\
    \end{align    }
    $$

    关于插值的更多内容，见 从零开始几何处理：函数拟合 - 知乎 (zhihu.com)

    插值系数搞定了，现在我们考虑：已经有了关键帧的骨骼姿态，究竟要插值什么东西，才能得到非关键帧的骨骼姿态，并且让非关键帧动画平滑且连续？
    一个非常简单的思路是，直接插值关节坐标。这在动作幅度小的时候问题不大，但是如果动作幅度大，不同的关节的运动速度是不同的，很容易插值出不自然的结果。

    另一个方法是插值变换矩阵，这涉及了正向运动学的内容。
    还可以插值部分关节坐标，其余的对逐帧进行逆向运动学的解算。

    正向动力学¶

    如果我们可以控制每个关节的坐标，那么也就控制了整个骨架的姿态，进而控制的角色模型。

    实际上，树上的每个关节节点存储的并不是它的绝对坐标，而是相对父节点的坐标，更精确的，是相对父节点的坐标变换矩阵。

    相对变换可以带来这些便利性：

    1. 更加符合直觉。一个关节的移动，可以带动整个子树。比如我们运动胳膊肘的时候，整个小臂也跟着动了。
    2. 可以简化动画的表示。通过这种依赖关系，如果需要确定整个角色的坐标，只需要确定根节点的坐标即可。另外，跟上一条类似，做手臂运动的动画不需要操作所有手臂关节，只需要操作肘关节。
    3. 可以更容易的进行动画数据压缩、动作融合等。这也是得益于上述优点，此处不详细论述。

    由于存储的是相对变换，那么计算关节坐标和计算关节坐标到世界坐标的变换便不再是显然的。

    正向运动学（Forward kinematics）正是为了解决这一问题：给定关节模型和每个节点相对父节点的变换矩阵，如何计算关节坐标到世界坐标的变换矩阵？

    关节模型¶

    这里我们看一下关节模型的具体定义。

    上图是三段骨骼。为了定义整个关节模型，我们需要储存：

    1. 每个关节所在骨骼的长度。
    2. 每个关节所在骨骼相对上一关节的旋转角度。
    3. 关节模型的根节点的坐标。

    为了引入空间变换，首先介绍骨骼动画涉及的几个坐标空间（坐标系）：

    1. 世界坐标系。顾名思义。树根的坐标代表了对应的物体在世界坐标系下的坐标。
    2. 关节坐标系。以某个关节为原点的坐标系，代表了树上某一节点的局部空间。
    3. 骨架（物体）坐标系。树根坐标为原点的坐标系。特别的，其实就是树根节点的关节坐标系。

    正向运动学就是求解关节坐标系到世界坐标系的变换。

    这样，再回看关节模型中储存的信息：

    1. 每个关节所在骨骼的长度：对应了当前关节坐标系到父关节坐标系的平移变换。
    2. 每个关节所在骨骼相对父关节的旋转角度：对应了当前关节坐标系到父关节坐标系的旋转变换。
    3. 关节模型的根节点的坐标：这对应了骨架坐标系到世界坐标系的平移变换。

    正向运动学¶

    回看上一张图。

    这里的根节点坐标是 \(x_0\) 。定义关节的末端影响器为这个关节伸出去的末端。如何计算末端影响器的世界坐标？

    设第 i 个关节对应的旋转矩阵是 \(R_i\)，平移矩阵是 \(T_i\) ，原点坐标 \(O=(0,0,0)\) ，那么：

    1. \(p_0\)的关节坐标是 \(O\) ，这也是在骨架坐标系的坐标，通过根节点的世界坐标进行变换，即对应的世界坐标是 \(x_{0} + O = x_0\)

    2. \(p_1\)的关节坐标是 \(O\) ，首先由 \(T_1\)进行一个平移，再由 \(R_1\) 进行旋转，得到它在父关节的关节坐标系坐标\(R_1T_1O\)，这也是它在骨架坐标系的坐标，所以对应的世界坐标是 \(x_{0}+ R_1T_1O\)

    3. \(p_2\)的关节坐标是 \(O\) ，同理可得它的世界坐标是 \(x_{0}+ R_1T_1R_2T_2O\)

    4. \(p_3\)同理，不过没在图上标注，就是最后一块骨骼的末端。世界坐标是 \(x_{0}+ R_1T_1R_2T_2R_3T_3O\)

    如果我们把 \(x_0\)也看做一个平移变换 \(T_0\)，再添加一个旋转变换 \(R_0=I\)，那么很容易发现，从只需要拿原点坐标乘平移、旋转矩阵，再乘父节点，一直乘到根节点，就可以得到关节的末端影响器的世界坐标。即
    $\(x_{i}= R_{root}T_{root}...R_{fa_i}T_{fa_i}R_iT_iO\)$

    所以，关节坐标系到世界坐标系的变换，就是根节点到该关节的变换矩阵之乘积。即

    \[M_{i-to-world}= R_{root}T_{root}...R_{fa_i}T_{fa_i}R_iT_i\]

    那么关节坐标系中任意点的坐标 \(p\)在世界坐标系中的坐标 \(p^{(w)}\)为
    $\(p^{(w)} = M_{i-to-world}\times p\)$

    另外注意到，我们可以通过树遍历一次性求出所有关节的变换矩阵

    这里的\(T_i\)实际上代表了刚刚的 \(R_iT_i\)。

    由此，我们可以求解末端影响器的世界坐标，并且可以求解关节坐标系和世界坐标系任意点坐标之间的对应。此即正向动力学。

    回到插值¶

    上述介绍中只涉及了平移和旋转，实际上有时还会涉及缩放，例如人物突然变大等等。

    所以，通过动画师编辑，我们可以得到每个骨骼在关键帧上的平移矩阵、旋转矩阵、缩放矩阵，这些都是相对于父节点的。只要这些变换矩阵确定了，人物的姿态就确定了。

    现在我们要插值非关键帧，除了插值关节的坐标，我们还可以插值每个骨骼的变换矩阵，从而得到平滑的结果。

    平移矩阵和缩放矩阵可以简单的通过线性插值得到，但是旋转矩阵不可以，因为：

    所以我们应该插值旋转角度，比如一些球面插值的方法。

    后记¶

    大多数情况下，骨骼的长度和大小都是不变的，我们只需要改变旋转角度，即\(R_i\)可能有变化，记为\(\theta_i\)
    此时，正向运动学可以看做给定旋转角度，求坐标，即求解 \(x =f(\theta)\)
    逆向运动学是求解 \(\theta =f^{-1}(x)\)，即给定空间坐标，求每块骨骼旋转角。
    下一章将介绍一些基本的逆向运动学求解算法。

    逆向动力学(IK)¶

    上一章介绍了如何通过关节模型来计算末端影响器的位置。

    更具体的，假设每块骨骼的长度不变，我们可以通过旋转骨骼来改变关节末端影响器的坐标，即 \(x =f(\theta)\) ，此为正向运动学。

    逆向运动学（Inverse kinematics）是求解相反的问题：给定末端影响器坐标，计算旋转的角度，即 \(\theta =f^{-1}(x)\)

    逆向运动学的需求十分常见，例如需要手去抓取物品。如果通过不断调节旋转角度来达成，那么会非常麻烦。

    有了逆向运动学，编辑关键帧会变的比较轻松。另外，对于插值非关键帧，我们也可以对关节坐标进行插值，然后对每一个非关键帧进行逆向运动学求解，这样的结果可能更加自然。

    但是，求解逆向运动学是比较困难的：

    另外，我们还有一些求解的约束，例如旋转角度范围等，例如膝盖不能翻到上面。

    这里就介绍一些比较经典的求解方法。

    循环坐标下降法（CCD）¶

    循环坐标下降法（CCD, Cyclic Coordinate Descent）是一种启发式的IK算法，也是目前游戏中求解 IK的标配算法。算法思路非常简单：

    如上图所示，就是每次选择某一个关节旋转一个角度，让末端影响器距离目标点位置更近。通过多次循环选择关节，最终得到结果。

    具体来说，就是每次旋转到当前关节与目标点的连线上。

    这个方法是可以收敛的，其实就是一种经典的优化方法，叫坐标下降法，每次只优化一个坐标，多次优化达到目的。

    求解时，需要注意一些旋转约束。

    基于优化的方法¶

    规定一个目标函数，让末端影响器的位置与目标位置的距离尽量近，并且满足约束，之后采用数值优化的方式求解。

    这里我们举一个简单的例子，只考虑距离尽量近。

    设优化目标
    $$ \theta^{*}= \arg \min_{\theta} \mathcal{L}(\theta) = \arg \min_{\theta} \frac{1}{2}(f(\theta) - q)^2$$

    其中 \(q\) 是目标位置， \(f(\theta)\)是末端影响器的位置。

    然后考虑采用梯度下降法
    $$ \theta_{t+1} = \theta_{t} - \alpha\nabla \mathcal{L}$$

    计算梯度

    \[ \nabla \mathcal{L} = (f(\theta) - q) \nabla f\]

    现在考虑末端影响器位置对关节 \(i\) 的旋转角度 \(\theta_i\) 的梯度。 \(\theta_i\)可以看作是一个三维的欧拉角，我们又可以把它拆成绕 x 轴、y 轴、z 轴转动的三个角，令其为\(\theta_{i,j}\) 。于是，这个关节可以看做三个小关节的组合，每个小关节只能绕一个轴转动。

    接下来我们只需要考虑对小关节的旋转角的偏导即可，即 \(\frac{\partial f}{\partial\theta_{i,j}}\)。

    现在考虑几何法，固定其它关节不动，只旋转这个小关节，那么有

    即，将末端影响器与当前关节连线，末端影响器会以它为半径做圆弧运动。

    于是可以得到

    \[\frac{\partial f}{\partial\theta_{i,j}} = | r_{i,j} \times (f(\theta) - o_i)|\]

    \(r_{i,j}\) 为第 \(i\) 个关节在世界坐标系下的旋转轴方向的单位向量 , \(o_i\)为第 \(i\)个关节在世界坐标系下的坐标。

    然后就可以用梯度下降法求解了。不过这里没有考虑各种约束，实际应用中会更复杂。

    逆向雅可比方法¶

    考虑 \(x =f(\theta)\)，我们求个微分
    $\(dx = \frac{\partial f}{\partial \theta}d\theta\)$

    这个式子的物理意义是，单位时间内线速度等于每个关节的角速度乘以每个关节的贡献。

    那么有

    \[d\theta = (\frac{\partial f}{\partial \theta})^{-1}dx\]

    然后，只需要选择一个很小的单位时间线速度，不断迭代即可

    \[\theta^{t+1} = \theta^{t} + (\frac{\partial f}{\partial \theta})^{-1}\Delta x\]

    f 是一个三元函数， \(\frac{\partial f}{\partial \theta}\) 其实是一个雅可比矩阵，如果像之前所述把角度拆成绕三个轴旋转的三个角，那么

    \[\frac{\partial f_j}{\partial\theta_{i,j}} = | r_{i,j} \times (f(\theta) - o_i)|\]

    所以我们可以计算出 \(J = \frac{\partial f}{\partial\theta}\) ，它的大小是 \(3\times3 N\) ，其中 \(N\)是关节数。

    现在的问题是求逆，可以用伪逆，即使用\((J^{\dagger}J)^{-1}J^{\dagger}\) 代替逆矩阵。

    由于可能有病态矩阵问题，可以采用一个控制系数让结果更鲁棒，即采用\((J^{\dagger}J + \lambda I)^{-1}J^{\dagger}\)代替逆矩阵。

    蒙皮与其他技术¶

    绑定¶

    之前的文章只介绍了如何操作骨骼，这篇文章介绍如何用骨骼带动皮肤。

    需要带动皮肤，首先需要指定骨骼与皮肤的对应关系。

    通常，绑定是针对一个标准姿势（静止姿势，rest pose）进行，也称之为 T-pose，因为人物的形态像个 T 字

    选择这种姿势是因为大部分姿势不会偏离它太大。

    我们需要对 T-pose 找到皮肤顶点与骨骼之间的对应，显然可以让一个顶点找到距离他最近的骨骼。

    大部分情况下，一个顶点只会被一个骨骼所控制，但是在关节的边界情况则可能会出现问题，因为不知道这个顶点究竟属于关节两侧的哪一个骨骼。

    所以一个方法是采用线性混合的方法，这个技术称之为顶点混合（Vertex blending）或者线性混合蒙皮（linear blend skinning）。这也是工业界中使用的技术。

    线性混合蒙皮¶

    假设对于节点 \(i\)，它在 T-pose 下的世界坐标是 \(v_i\)，有 \(m\)个骨骼控制它，第 \(j\)个骨骼在当前时刻的变换矩阵是 \(M^{(t)}_j\)，权重是 \(w_{i,j}\)，那么最终的结果的世界坐标 \(v^{'}_{i}\)是

    \[v^{'}_{i} =\sum^{m}_{j=1}w_{i,j}M^{(t)}_jv_{i}=(\sum^{m}_{j=1}w_{i,j}M^{(t)}_j)v_i\]

    其中 \(\sum_{j}w_{i,j} = 1\) 。通常，m 至多为 4。

    所以实际上是对每个骨骼的变换矩阵进行插值，插值系数交由美工来调整。

    注意到，旋转变换直接插值后并不是旋转变换，所以需要采用球面插值公式。

    这里需要用到四元数来表示旋转，因为它比矩阵表示和欧拉角表示更容易插值。带权球面插值需要用到 Dual quaternion skinning 算法，它能修复如下结果

    蒙皮的实现¶

    我们先来把\(M^{(t)}_j\)写出来。在介绍正向运动学时，我们推导了关节坐标系到世界坐标系的变换矩阵

    \[M_{j-to-world}= R_{root}T_{root}...R_{fa_j}T_{fa_j}R_jT_j\]

    其实在变换姿态的时候，我们不会考虑世界坐标，只会考虑骨架坐标，即

    \[M_{j-to-skeleton}= (R_{root}T_{root})^{-1}M_{j-to-world}\]

    假设 T-pose 中关节坐标系到骨架坐标系的变换矩阵是 \(M_{j-to-skeleton}^{(0)}\) ，当前时刻关节坐标系到骨架坐标系的变换矩阵是\(M_{j-to-skeleton}^{(t)}\)

    故有

    \[M_{j}^{(t)} = M_{j-to-skeleton}^{(t)}(M_{j-to-skeleton}^{(0)})^{-1}\]

    注意到 \(M_{j-to-skeleton}^{(0)}\)是绑定时就有的，也就是说它是附着于骨架（或者说对应的模型）的，只有\(M_{j-to-skeleton}^{(t)}\)需要实时的去计算。

    所以，当美工完成骨架与皮肤的绑定后，\((M_{j-to-skeleton}^{(0)})^{-1}\) 就唯一确定了，它与骨架的姿态有关。所以带蒙皮的骨架文件中会连带存储每块骨骼的该矩阵，以及每个皮肤顶点被影响的骨骼 id，还有权重。

    在运行中的每一帧，对于每段骨骼的变换矩阵，\((M_{j-to-skeleton}^{(0)})^{-1}\) 会直接读取，\(M_{j-to-skeleton}^{(t)}\)会分解成平移、旋转、缩放三个部分，分别通过 CPU 插值计算得出，由于骨骼一般只有几十根到一两百根，所以开销并不大。之后相乘得到每根骨骼的变换矩阵 \(M_{j}^{(t)}\)

    为了变换皮肤上每个顶点的坐标，需要把每根骨骼的变换矩阵放到 GPU 里，使用 vertex shader 变换得到新位置。因为要放到 GPU 里，为了让数据传递的开销别那么大，一般一个顶点只会被至多 4 根骨骼影响，这也很够用了。如果是用 CPU 去做变换，那就没必要遵循这个限制。

    其它技术¶

    这里就粗略介绍一下其他技术。

    动作捕捉。美工手工编辑关键帧耗时耗力，而且很难做出自然的效果。目前业界的成熟方案是采用动作捕捉+后期修正的方法，缺点是价格昂贵而且难以编辑（一般不好就重新拍摄了）。动作捕捉获取的骨骼动画需要通过采样来离散化，从而进行存储。

    动画数据。按照时间存储变换矩阵，变换矩阵会拆分为旋转（R），平移（T），缩放（S）三个通道分别存储。这样的好处是可以分别进行压缩与插值。实际上现代游戏越来越大的安装包，几乎都存储的是贴图和动画数据，压缩动画数据非常重要。

    换装系统。换装就是骨架不动，换皮肤，即 mesh。服装有时也有自己的骨架，为了适配有两种解决方法：人物骨架集合所有的服装骨架、服装骨架作为子骨架与人物骨架动态拼接。前者适配方便但是骨骼太多，后者更灵活但是动画管理复杂。

    动作相关。动作和动画不是一个东西，动画是一个个小片段，动作是动画的组合。

    动作融合。一个角色同时播放多个动画，需要做合适的过渡，比如从站立状态变成跑步状态，这样更自然。简单的方法是直接将不同动画对某个骨骼的变换矩阵做一个混合，但是实际使用上会遇到各种各样的问题。

    状态机。状态机描述了一套完整的动作。例如从战斗到死亡，技能动画、受伤动画、攻击动画、死亡动画看做状态机中的每个点，点之间通过状态转移边连接起来，边上有过渡的条件以及时间等等信息。一套动作就是状态机上的一条路径。当动作变的复杂时，整个状态机也会异常庞大，难以维护。
